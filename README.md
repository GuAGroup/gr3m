# GR3M Protocol 🛡️

**GR3M** — это высокопроизводительный асинхронный сетевой туннель, разработанный для обхода систем глубокого анализа трафика (DPI). Протокол использует динамическую маскировку под стандартные WebSocket/HTTP запросы, обеспечивая при этом криптографическую защиту.

## ✨ Ключевые особенности

* **Perfect Forward Secrecy (PFS):** Обмен эфемерными ключами на базе кривой **X25519** для каждой сессии. Перехваченный ранее трафик невозможно расшифровать даже при компрометации долгосрочных секретов.
* **Защита от Replay-атак:** Система монотонных счетчиков (Sequence Numbers/Nonce) предотвращает повторную отправку перехваченных пакетов.
* **Smart Padding (Length Hiding):** Каждый пакет дополняется случайным количеством байт (до 256), что делает невозможным статистический анализ типа трафика по размеру фреймов.
* **Remote DNS:** Все DNS-запросы выполняются на стороне сервера. Это исключает утечки DNS у клиента и позволяет обходить блокировки на уровне имен.
* **Hysteria Masking (Decoy):** В случае ошибки авторизации или попытки сканирования, сервер маскируется под легитимный ресурс (например, Wikipedia), перенаправляя трафик.
* **Auto-Reconnect & Peer Selection:** Клиент автоматически замеряет задержку (RTT) до всех доступных узлов и выбирает оптимальный маршрут.

## 🚀 Быстрый старт

### Требования
* **Go** 1.22+ (рекомендуется 1.26+)
* **Docker** & **Buildx** (для контейнеризации)

### Сборка
Проект использует BuildKit для быстрой и безопасной сборки:
```bash
# Подготовка зависимостей
go mod tidy

# Сборка образа
DOCKER_BUILDKIT=1 docker build -t gr3m-app .

```

### Настройка (`config.json`)

Создайте файл конфигурации в корне проекта:

```json
{
  "mode": "client",
  "listen_addr": "0.0.0.0:8080",
  "peers": [
    {"name": "Main-Node", "addr": "1.2.3.4:8080"}
  ],
  "socks_addr": "127.0.0.1:1080",
  "decoy_url": "wikipedia.org:80"
}

```

## 🛠️ Использование

### Запуск сервера (на VPS)

```bash
docker run -d \
  --name gr3m-server \
  -p 8080:8080 \
  --restart always \
  gr3m-app

```

### Запуск клиента (на локальной машине)

```bash
docker run -d \
  --name gr3m-client \
  -p 1080:1080 \
  gr3m-app

```

*Настройте вашу систему или браузер на SOCKS5 прокси: `127.0.0.1:1080`.*

## 🏗️ Архитектура

* `/protocol`: Реализация крипто-слоя (ChaCha20-Poly1305), компрессии и защиты целостности.
* `/core`: Логика Handshake, обмена ключами и управления состоянием сессии.
* `/client`: SOCKS5-сервер, асинхронный мост и механизм Auto-Reconnect.
* `/server`: Релей-движок, динамическое проксирование и система маскировки.

## 🔒 Модель угроз (Threat Model)

1. **Конфиденциальность:** Весь Payload зашифрован ChaCha20-Poly1305 (AEAD).
2. **Целостность:** Любое изменение бита в пакете приводит к немедленному разрыву соединения.
3. **Анализ трафика:** Рандомизация длин пакетов и имитация HTTP-заголовков скрывают природу протокола.
4. **Forward Secrecy:** Ключи сессии генерируются "на лету" и не сохраняются на диске.
