  # GR3M Protocol
  
  *Высокопроизводительный асинхронный туннель для обхода DPI*
</div>

**GR3M** — высокопроизводительный асинхронный сетевой туннель, разработанный для обхода систем глубокого анализа трафика (DPI). Протокол имитирует легитимный HTTPS/WebSocket трафик, используя отпечатки реальных браузеров и современные методы криптографии для обеспечения максимальной скрытности и безопасности.

## ✨ Ключевые особенности

* **uTLS Fingerprinting:** Имитация TLS-handshake браузера Chrome для обхода сенсоров DPI, которые блокируют нестандартные TLS-соединения.
* **Server Identity Binding:** Защита от атак типа "человек посередине" (MITM) через проверку хеша публичного ключа сервера (Pinning).
* **Perfect Forward Secrecy (PFS):** Использование эфемерных ключей на базе кривой **X25519** для каждой сессии.
* **AEAD Encryption:** Шифрование данных с использованием **ChaCha20-Poly1305**, гарантирующее конфиденциальность и целостность каждого пакета.
* **Smart Padding:** Динамическое заполнение пакетов случайными данными для скрытия паттернов трафика и реального размера передаваемой информации.
* **Hysteria Masking:** Система маскировки под стандартный веб-ресурс (Decoy) при попытках активного зондирования или некорректном хендшейке.
* **Auto-Reconnect & Latency Selection:** Автоматический выбор узла с минимальной задержкой (RTT) и поддержание стабильного соединения.

## 📑 Спецификация протокола (Wire Format)

Структура пакета перед шифрованием:

| Смещение | Размер | Описание |
| --- | --- | --- |
| 0 | 8 байт | **Nonce** (uint64, защита от атак повтора) |
| 8 | 4 байта | **Stream ID** (идентификатор потока) |
| 12 | 4 байта | **Payload Length** (размер полезной нагрузки) |
| 16 | N байт | **Payload** (сжатые и зашифрованные данные) |
| 16+N | M байт | **Random Padding** (случайное заполнение) |

## 🚀 Сборка и установка

### Требования

* Go 1.22+ (рекомендуется 1.26+)
* Docker & Buildx

### Сборка

```bash
go mod tidy
DOCKER_BUILDKIT=1 docker build -t gr3m-app .

```

## 🛠️ Настройка

### Генерация ключей

Для обеспечения безопасности Identity Binding используйте утилиту `keygen`:

```bash
go run cmd/keygen/main.go

```

### Пример config.json (Сервер)

```json
{
  "mode": "server",
  "listen_addr": "0.0.0.0:8080",
  "private_key": "ВАШ_ПРИВАТНЫЙ_КЛЮЧ_HEX",
  "decoy_url": "wikipedia.org:80"
}

```

### Пример config.json (Клиент)

```json
{
  "mode": "client",
  "socks_addr": "127.0.0.1:1080",
  "peers": [
    {
      "name": "Primary-Node",
      "addr": "IP_СЕРВЕРА:8080",
      "pub_key": "ХЕШ_ПУБЛИЧНОГО_КЛЮЧА_ИЗ_ГЕНЕРАТОРА"
    }
  ]
}

```

## 🔒 Безопасность

1. **Конфиденциальность:** Весь трафик зашифрован, чтение данных третьими лицами невозможно.
2. **Аутентификация:** Клиент доверяет только серверу с конкретным публичным ключом.
3. **Скрытность:** Использование uTLS и HTTP-заголовков делает трафик неотличимым от посещения обычного сайта.
4. **Устойчивость:** Любая модификация данных в канале приводит к отбрасыванию пакета благодаря AEAD.
